Intern Learning Roadmap

Day 1 — Topics & Libraries to Explore

A starter reading list to get familiar with the foundations and the Computer Vision / ML stack. Take your time — focus on understanding concepts, not memorizing syntax.

Duration: ~1 week Track: Computer Vision / ML Level: Beginner-friendly
Phase 1

Foundations

Core skills that every developer needs. Aim to finish in about a week.

Py

Python basics

  • virtualenv / pip — isolated environments & package install
  • Type hints (def foo(x: int) -> str:)
  • @dataclass for simple data containers
  • File I/O — reading & writing files
  • The requests library for HTTP calls
JS

JavaScript / Node basics

  • ES modules (import / export)
  • async / await & Promises
  • npm commands and package.json structure
Git

Git & GitHub

  • Branches & pull requests
  • Merge vs. rebase — when to use which
  • git lfs — handling large binary files in a repo
API

REST APIs

  • HTTP verbs: GET, POST, PUT, DELETE
  • Status codes (200, 201, 400, 401, 404, 500)
  • JSON request & response bodies
  • JWT auth — how tokens work, where they go in the header
$_

Linux / CLI

  • Bash essentials: ls, cd, cat, grep, cp, mv
  • Environment variables & .env files
▷

Tool to learn

  • Postman or Thunder Client (VS Code extension) — for sending requests to APIs and inspecting responses
Phase 2 — Pick a track
Track A

Computer Vision / ML

A broad survey of the field — image processing, deep learning, detection, classification, segmentation, OCR, and more.

★ Look for the yellow tags. Items tagged RackTrack are what our project actually uses — learn the broad concepts first, then go deeper on the tagged ones since you'll be working with them day-to-day.
1

Image processing fundamentals

  • OpenCV (opencv-python): reading frames, color spaces (RGB / BGR / HSV / grayscale), resizing, cropping, drawing
  • NumPy — images as multi-dimensional arrays, slicing, masking, vectorized operations
  • PIL / Pillow — basic image I/O, format conversion
  • Filters & transforms: blur, sharpen, edge detection (Canny, Sobel), thresholding, morphological ops
  • Geometric transforms: rotation, affine & perspective warps
2

Machine learning & deep learning basics (concepts, no heavy math)

  • Supervised vs. unsupervised learning
  • Training / validation / test splits, overfitting & regularization
  • Neurons, layers, activations (ReLU, sigmoid, softmax)
  • What a CNN is — convolutions, pooling, feature maps
  • Key terms: epoch, batch, loss, learning rate, confidence threshold, backpropagation
  • Frameworks: PyTorch, TensorFlow / Keras, JAX
  • Hugging Face transformers & datasets — the modern ecosystem
3

Image classification

  • Classic CNN architectures: LeNet, AlexNet, VGG, ResNet, Inception
  • Modern architectures: EfficientNet, MobileNet, ConvNeXt
  • Vision Transformers: ViT, Swin Transformer, DeiT
  • Multimodal: CLIP, DINOv2 (image + text understanding)
  • Transfer learning & fine-tuning a pre-trained backbone
4

Object detection

  • Two-stage detectors: R-CNN, Fast R-CNN, Faster R-CNN
  • One-stage detectors: SSD, RetinaNet, EfficientDet
  • YOLO family: YOLOv3 → v5 → YOLOv8 (Ultralytics) → YOLOv9 / v10 / v11
  • Transformer-based: DETR, DINO-DETR, RT-DETR
  • Concepts: anchor boxes, IoU, NMS (Non-Max Suppression), mAP metric
  • Reading model output: bounding boxes, class IDs, confidence scores
5

Image segmentation

  • Semantic vs. instance vs. panoptic segmentation
  • U-Net, Mask R-CNN, DeepLab
  • SAM (Segment Anything Model) — Meta's foundation model for segmentation
6

OCR (Optical Character Recognition)

  • Tesseract — classic open-source OCR engine
  • EasyOCR — Python library, 80+ languages out of the box
  • PaddleOCR — fast, modern, multi-language
  • TrOCR — transformer-based OCR by Microsoft
  • Use cases: documents, signs, labels, license plates, handwriting
7

Object tracking & video

  • SORT, DeepSORT, ByteTrack, BoT-SORT — multi-object tracking
  • Optical flow basics
  • Action recognition: SlowFast, I3D
8

Datasets & benchmarks

  • ImageNet, COCO, PASCAL VOC, Open Images
  • Labeling tools: LabelImg, CVAT, Roboflow, Label Studio
  • Data augmentation: Albumentations, torchvision.transforms
9

Evaluation metrics

  • Classification: accuracy, precision, recall, F1, confusion matrix, ROC / AUC
  • Detection: IoU, mAP @ 0.5 / 0.5:0.95
  • Segmentation: Dice coefficient, pixel accuracy
10

Deployment & optimization

  • Model formats: ONNX, TorchScript, TensorFlow Lite, Core ML
  • Acceleration: TensorRT, OpenVINO, CUDA / cuDNN
  • Quantization & pruning (shrinking models for edge devices)
  • Serving: TorchServe, Triton Inference Server, FastAPI
11

Generative & foundation models (awareness level)

  • GANs: StyleGAN
  • Diffusion: Stable Diffusion, DALL·E
  • Multimodal LLMs: GPT-4V, Gemini, LLaVA
✦ By the end of this track, you should be able to explain
  • What an image looks like to a computer (pixels, channels, arrays)
  • The difference between classification, detection, segmentation, and OCR
  • What a pre-trained model is and why we use one (transfer learning)
  • How bounding boxes, IoU, and confidence scores work
  • The difference between training and inference
  • How to read a model's evaluation metrics and judge if it's "good enough"
Take your time, take notes, and ask questions when something doesn't click. You've got this.
