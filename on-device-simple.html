On-Device Inference — Technical Summary

Porting the 9-model pipeline from server to mobile.

Date: 13 May 2026 Device: Samsung Galaxy A35 5G · arm64-v8a Status: Feasibility confirmed

1.Issue

  • Architecture: 9 models (YOLOv8 detectors + EfficientNet classifier) hosted server-side; mobile client invokes them via HTTPS per scan.
  • Footprint: 467 MB combined FP32 weights — non-shippable inside an Android APK.
  • Dependency: Every scan requires connectivity. Target environments (data centres, server rooms) have poor or no mobile signal.
  • Cost: Per-scan network round-trip plus recurring server GPU compute that scales linearly with usage.

2.Solution

  • Convert all models to ONNX (cross-platform inference format).
  • Apply dynamic INT8 post-training quantization: weights cast from FP32 → INT8, reducing size ~4× and improving CPU throughput.
  • Execute locally via onnxruntime-web (WebAssembly) inside the existing Capacitor app — no server, no network.
  • Expected trade-off: < 1% accuracy degradation for the architectures in use.

On-device benchmark (single model, 640×640 input, 10 iterations):

  • Load time: 953 ms (cold)
  • Inference latency: 6397 ms median, ±30 ms stability
  • Output shape: [1, 5, 8400] — YOLO head fires correctly
  • Memory: stable, no OOM observed across the 9-model run

Validated: Quantization preserves output integrity (0.39% mean Δ). Full pipeline executes locally with zero server traffic. APK size remains within Play Store distribution limits (156 MB bundled).

Gap: WASM runtime caps per-model latency at ~6 s — unsuitable for the live-camera flow. Resolution: migrate to onnxruntime-android with NNAPI / GPU delegation. Documented speed-up of 10–20× projects to 300–600 ms per inference. Estimated effort: 2–3 engineering days.

Conclusion: On-device inference is technically feasible. Size, accuracy, and execution are validated. Remaining work is a runtime swap, not an architectural change.
